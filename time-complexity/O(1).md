# O(1) Time Complexity

## Definition
An algorithm is **O(1)** if it takes **constant time**, regardless of input size `n`.

- Execution time does **not grow** with `n`
- Operations are fixed and independent of input size

---

## Core Understanding

Ask:
- Does this operation depend on `n`?
  - If **NO** → O(1)

Even if input is huge, operation count stays the same.

---

## Basic Patterns

### 1. Direct Access (Array)
```cpp
int x = arr[5];
```

---

### 2. Assignment
```cpp
int x = 10;
```

---

### 3. Arithmetic Operations
```cpp
int sum = a + b;
int mul = a * b;
```

---

### 4. Comparison
```cpp
if(a > b) {
    cout << a;
}
```

---

### 5. Swap
```cpp
swap(a, b);
```

---

## Important Data Structure Operations

### 6. Stack
```cpp
stack<int> st;

st.push(5);
st.pop();
int x = st.top();
```

---

### 7. Queue / Deque
```cpp
queue<int> q;

q.push(10);
q.pop();
int x = q.front();
```

---

### 8. Hash Structures (Average Case)
```cpp
unordered_map<int,int> mp;

mp[5] = 10;
int x = mp[5];
```

---

### 9. Vector (End Operations)
```cpp
vector<int> v;

v.push_back(10);
v.pop_back();
int x = v.back();
```

---

## Tricky Cases (Important)

### Case 1: Looks Small but Not O(1)
```cpp
for(int i = 0; i < n; i++) {
    cout << i;
}
```
This depends on `n` → NOT O(1)

---

### Case 2: Fixed Loop (Still O(1))
```cpp
for(int i = 0; i < 100; i++) {
    cout << i;
}
```
Runs fixed 100 times → O(1)

---

### Case 3: Multiple Constant Operations
```cpp
int a = arr[0];
int b = arr[1];
int c = arr[2];
```
Still constant work

---

### Case 4: Conditional Branching
```cpp
if(x > 10) cout << "A";
else cout << "B";
```
Only one branch executes → O(1)

---

## ⚠️ Subtle Points

### 1. Input Reading Is Not O(1)
```cpp
cin >> x;
```
Single variable → O(1)

```cpp
for(int i = 0; i < n; i++) cin >> arr[i];
```
Depends on `n`

---

### 2. Function Calls
```cpp
int f() {
    return 5;
}
```
If function does constant work → O(1)

---

### 3. Access vs Search
```cpp
int x = arr[5];   // O(1)
```

Searching is different:
```cpp
// searching element in array → depends on n
```

---

## Common Mistakes

### Mistake 1: Assuming Small Code = O(1)
```cpp
for(int i = 0; i < n; i++) {}
```
Even if code looks short → depends on `n`

---

### Mistake 2: Hidden Iteration
```cpp
v.size()
```
This is O(1), but:

```cpp
count(v.begin(), v.end(), x)
```
This is not constant (iterates internally)

---

### Mistake 3: Misunderstanding Hashing
```cpp
unordered_map<int,int> mp;
mp[x]++;
```
Treated as constant per operation (average)

---

## Key Observations

- Constant operations remain constant regardless of input size
- Fixed-size loops are O(1)
- Data structure operations at ends (stack, queue, vector) are constant

---

## Recognition Strategy

To confirm O(1), check:
1. Does it avoid loops depending on `n`?
2. Does it avoid recursion depending on `n`?
3. Is the number of operations fixed?

If yes → O(1)

---

## Summary

- Direct access → O(1)
- Basic operations → O(1)
- Stack / Queue operations → O(1)
- Hash operations (average) → O(1)
- Fixed work only → O(1)

**Key idea:**  
If input size changes but work does not → O(1)
